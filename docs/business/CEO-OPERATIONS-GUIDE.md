## 2. Структура проектов в Claude

Claude поддерживает **проекты** — изолированные рабочие пространства с собственными инструкциями.
Рекомендую создать **5 проектов**:

### Проект 1: "Operating Officer" (уже существует)
**Где:** Cowork
**Для чего:** Стратегия, приоритизация, архитектура, координация между проектами.
**Когда открываешь:** Утренний обзор задач, планирование спринта, разбор идей, создание спеков.
**Инструкции (краткая суть):**
```
Role: COO и системный архитектор.
Опирается на: p0-rules.md, DISPATCH_RULES.md, STATUS.md.
Не пишет код — проектирует и координирует.
```

### Проект 2: "Admin Panel Dev"
**Где:** Claude Code (терминал)
**Для чего:** Разработка admin panel — React/Next.js, Supabase, UI компоненты.
**Когда открываешь:** Имплементация конкретной задачи из спека.
**Инструкции (рекомендуемые):**
```
Role: Frontend developer для Shishka admin panel.
Stack: Next.js 14, Supabase, Tailwind, shadcn/ui.
Перед работой: читай docs/plans/QUEUE.md, бери верхнюю ready задачу.
Перед коммитом: обновляй CURRENT.md и STATUS.md.
Стиль: TypeScript strict, компоненты в app/(dashboard)/,
серверные actions в lib/actions/.
НЕ трогай другие проекты (web, app).
```

### Проект 3: "Chef & Menu R&D"
**Где:** Cowork (с MCP shishka-chef)
**Для чего:** Работа с рецептами, BOM, нутриентами, тестирование блюд, поиск по кулинарной базе.
**Когда открываешь:** Разработка нового блюда, калькуляция себестоимости, аудит меню.
**Инструкции:**
```
Role: Chef R&D assistant.
Используй MCP: search_products, get_bom_tree, calculate_cost,
calculate_nutrition, search_knowledge.
Документируй результаты тестов в docs/kitchen/test-results/.
Новые продукты — через create_product с confirmed=true только
после одобрения CEO.
Язык: рецепты на русском, product_code на английском.
```

### Проект 4: "Business Docs & Analysis"
**Где:** Cowork
**Для чего:** Создание документов (SOP, отчеты, презентации), финансовый анализ, подготовка к встречам.
**Когда открываешь:** Нужен docx/xlsx/pptx/pdf артефакт.
**Инструкции:**
```
Role: Business analyst и документ-мейкер.
Скиллы: docx, xlsx, pptx, pdf — всегда читай SKILL.md перед созданием.
Бренд: Shishka Healthy Kitchen, цвета: зеленый (#4CAF50), белый.
Сохраняй финальные файлы в доступную папку.
Язык: русский для внутренних, английский для внешних документов.
```

### Проект 5: "Delivery Website" (создать когда будет ready)
**Где:** Claude Code
**Для чего:** Разработка клиентского сайта для заказа доставки.
**Когда открываешь:** После утверждения дизайна и спека.

## 3. Cowork vs Code — когда что

| Ситуация | Инструмент | Почему |
|----------|------------|--------|
| Разбор идеи, приоритизация | **Cowork** | Видит файлы + MCP, может анализировать и создавать задачи |
| Написание спека/ТЗ | **Cowork** | Создает docs/plans/spec-*.md, логирует в MC |
| Работа с рецептами и BOM | **Cowork** | MCP shishka-chef доступен |
| Создание docx/xlsx/pptx | **Cowork** | Есть скиллы для документов |
| Имплементация UI компонента | **Code** | Git, LSP, тесты, итеративная разработка |
| Миграция БД, SQL | **Code** | Supabase CLI, тестирование запросов |
| Баг-фикс | **Code** | Дебаг, git blame, быстрые правки |
| Код-ревью | **Code** | git diff, контекст всего проекта |
| Утренний обзор (что делать сегодня) | **Cowork** | list_tasks + анализ приоритетов |
| Документация архитектуры | **Cowork** | Mermaid, markdown, vault/Architecture/ |

### Правило большого пальца:
> **Cowork** = мозги (думать, планировать, документировать, работать с данными)
> **Code** = руки (писать код, коммитить, деплоить)

## 5. Как оформлять эпики (Initiative → Tasks)

Текущая проблема: в Mission Control задачи плоские, без иерархии. Решение — использовать **initiatives + parent_task_id + tags**.

### Структура:

```
business_initiative (таблица)
  └── business_task (parent_task_id = null, tag: "epic")
       ├── business_task (parent_task_id = epic_id)
       ├── business_task (parent_task_id = epic_id)
       └── business_task (parent_task_id = epic_id)
```

### Конвенция именования:

**Initiative** (крупный проект, 3+ доменов):
`"Initiative: Kitchen Production System"`

**Epic task** (группа связанных задач в одном домене):
`"[EPIC] Barcode & Batch Tracking"`

**Обычная задача** (конкретное действие):
`"Barcode printer integration — connect physical printer to batch label generation"`

### Теги для фильтрации:
- `epic` — задача-эпик (группирующая)
- `blocked` — зависит от другой задачи
- `needs-spec` — нужен спек перед работой
- `needs-hardware` — требует физическое оборудование
- `quick-win` — можно сделать за 1 сессию

## 6. Рабочий процесс CEO (ежедневный)

### Утро (15 мин в Cowork "Operating Officer"):
1. `list_tasks(status="inbox")` — что нового?
2. Триаж: inbox → backlog / in_progress / отклонить
3. Приоритизировать: что делаем сегодня?
4. Если нужен спек — написать в docs/plans/

### Работа (в соответствующем проекте):
- Рецепт? → "Chef & Menu R&D"
- Код? → "Admin Panel Dev" в Code
- Документ? → "Business Docs"
- Идея пришла? → вернуться в "Operating Officer", создать задачу

### Вечер (5 мин):
- Обновить статусы задач
- Записать что сделано в STATUS.md
