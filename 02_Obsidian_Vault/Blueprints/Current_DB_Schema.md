# CURRENT LIVE DATABASE SCHEMA (Supabase Mirror)

This blueprint documents the active relationships in the `public` schema. It serves as the single source of truth for all Data Sync agents.

## 🗺️ Entity Relationship Diagram (ERD)

```mermaid
erDiagram
    NOMENCLATURE ||--o{ BOM_STRUCTURES : "is parent"
    NOMENCLATURE ||--o{ BOM_STRUCTURES : "is ingredient"
    NOMENCLATURE ||--o{ RECIPES_FLOW : "has recipe steps"
    EQUIPMENT ||--o{ RECIPES_FLOW : "used in steps"
    EQUIPMENT ||--o{ CAPEX_ASSETS : "is financial asset"
    FIN_SUB_CATEGORIES ||--o{ CAPEX_TRANSACTIONS : "mapped to"
    CAPEX_ASSETS ||--o{ CAPEX_TRANSACTIONS : "has records"

    NOMENCLATURE {
        uuid id PK

        text name
        text type "dish, good, modifier"
        uuid syrve_id
        text base_unit
    }

    BOM_STRUCTURES {
        uuid id PK
        uuid parent_id FK
        uuid ingredient_id FK
        numeric quantity_per_unit
        numeric yield_loss_pct
    }

    EQUIPMENT {
        uuid id PK
        text equipment_code UK
        text name
        text category
    }

    RECIPES_FLOW {
        uuid id PK
        text product_code FK
        text equipment_id FK "joins on code or uuid"
        text step_description
        integer step_order
    }

    CAPEX_ASSETS {
        uuid id PK
        uuid unit_id FK "links to equipment.id"
        text asset_name
        numeric initial_cost
    }
```

## 🛠️ Data Sync Logic
1. **Mesh Construction**: Notes are generated by querying `nomenclature` + `bom_structures`.
2. **Bi-directional Linking**:
   - `## Components` -> `bom_structures.ingredient_id`
   - `## Used In` -> `bom_structures.parent_id`
3. **P0 Integrity**: All links rely on `product_code` or `UUID`. Legacy table dependencies have been eliminated.
